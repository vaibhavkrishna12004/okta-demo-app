Implementation of Identity and Access Management using Okta

 Project Overview
This project demonstrates the implementation of *Identity and Access Management (IAM)* using Okta. It focuses on secure authentication, user management, and application access control using modern security protocols.

The project simulates a real-world environment where users securely log in to applications using *Single Sign-On (SSO)* and *OpenID Connect (OIDC)*.



 Objectives
- Implement secure user authentication using Okta  
- Configure Single Sign-On (SSO)  
- Demonstrate OpenID Connect (OIDC) authentication  
- Manage users and application access  
- Understand real-world IAM workflows  



 Tools & Technologies
- Okta (Identity Provider)  
- OAuth 2.0  
- OpenID Connect (OIDC)  
- Secure Web Authentication (SWA)  
- Web Browser (for testing)  



 Features Implemented
- User creation and management  
- Application integration in Okta  
- SSO (Single Sign-On) setup  
- OIDC-based authentication flow  
- Role-based access control (RBAC)  
- Secure login simulation  



 Implementation Steps
1. Created Okta Developer Account  
2. Configured users and groups  
3. Created applications (SWA and OIDC)  
4. Assigned users to applications  
5. Configured authentication settings  
6. Tested login and access flow  



Authentication Flow (OIDC)
1. User clicks on application  
2. Redirected to Okta login page  
3. User enters credentials  
4. Okta authenticates user  
5. Token (ID Token / Access Token) is generated  
6. User is redirected back to application  


 Results
- Successfully implemented secure authentication using Okta  
- Demonstrated SSO and token-based authentication  
- Simulated real-world IAM system without full backend
